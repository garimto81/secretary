## 분류 규칙

### project_id 결정 기준
- 메시지에 프로젝트 키워드가 명확히 등장하면 해당 프로젝트
- 키워드가 없어도 문맥상 특정 프로젝트와 관련되면 해당 프로젝트
- 기존 규칙 매칭 결과(rule_hint)가 있으면 참고 (단, LLM 판단이 우선)
- 어느 프로젝트에도 해당하지 않으면 null

### needs_response 기준
- 질문, 요청, 멘션이 있으면 true
- 정보 공유, 일반 대화는 false
- 긴급 키워드("급", "빨리", "urgent")가 있으면 true

### intent 분류
- **질문**: "어떻게", "무엇", "왜", "?" 포함 시
- **요청**: "해줘", "부탁", "처리" 등 행동 요청
- **정보공유**: "알림", "공유", "참고" 등 정보 전달
- **잡담**: 인사, 감사, 일상 대화

### confidence 기준
- 0.9 이상: 명확한 키워드 + 문맥 일치
- 0.7~0.9: 키워드 있지만 모호함
- 0.5~0.7: 추론 필요
- 0.5 미만: 불확실

## 응답 형식

반드시 JSON만 출력하세요. 설명 없이 아래 형식만 반환:

```json
{{
  "project_id": "secretary",
  "needs_response": true,
  "intent": "질문",
  "summary": "일일 리포트 실행 방법 문의",
  "confidence": 0.85,
  "reasoning": "secretary 키워드 + 질문 형식"
}}
```

/no_think 태그를 사용하여 빠르게 응답하세요.
